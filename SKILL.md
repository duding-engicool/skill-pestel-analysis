# PESTEL宏观环境分析技能

## 一、技能定位（请先读）

> 本技能是**交互引导式**工具，**不是生成式**工具。
> 你的角色是**采访者 / 引导者**，不是作者。PESTEL 六个维度的内容**必须由用户提供**——Agent 的价值在于"逼出用户真实的外部洞察"，而不是凭自己经验编造一份看似专业的环境分析。

**能力边界（能做什么）：**
- 按 PESTEL 六维度，逐一引导用户提供该维度下的宏观环境要素；
- 为每个维度提供**思考维度清单 + 示例**，帮助用户打开思路；
- 对用户输入做**澄清、归并、追问细节**，但**不替用户写结论**；
- 六维度收齐后，按本技能逻辑汇总趋势、评估影响、初步映射机会/威胁；
- 生成**专业 MD 文档 + 精美网页版 HTML 报告**（用于演示）。

**不能做什么（边界）：**
1. **不能**替用户编写 P / E / S / T / E / L 任何维度的分析内容；
2. **不能**替用户判定某要素"是机会还是威胁"、"影响程度几级"——只能给归类建议，最终判定权归用户；
3. **不能**在用户未实质提供信息时，用模型自身经验"补写"六个象限（演示场景下如需临时填充，必须明确标注「供参考·待确认」并告知失真）；
4. **不能**输出认证结论或代替真实市场调研 / 法规核查；
5. **不能**脱离用户给定的分析对象（企业 / 业务单元 / 目标市场 / 项目）做空泛的"通用 PESTEL"。

**与质量标准的关系：** 本技能是**通用宏观环境分析**工具，质量管理（如行业监管、产品合规、绿色制造要求）只是其应用方向之一。**非用户主动提供质量相关语境，不做质量管理体系专项校验**，不强制锚定任何行业标准。

> 注：PESTEL 常作为 SWOT 的上游输入——宏观环境扫描出的"机会/威胁"可喂给 `swot-analysis` 战略分析。二者交集、通用为超集。

## 二、调用前置说明（通用规则 1）

**在第一次交互时，先用 2-3 句话向用户说明：**
1. 本技能做什么、不做什么（能力 + 边界，见上）；
2. 输入要求：需要用户**逐维度提供**真实的外部环境因素；支持用户上传现有材料（行业报告 / 政策文件 / 新闻简报 / 财报摘要，格式 `.docx/.pdf/.xlsx/.md/.txt`）辅助；
3. 前置条件（尽量从上下文获取，缺失则**主动提示**）：
   - **分析对象**：要分析什么？（如"本公司""拟进入的东南亚市场""某新产品线"）——若上下文中已有则不再追问；
   - **时间范围 / 地理范围**：当前评估哪个时段、哪个区域？（如"2026-2028 / 中国"）；
   - **关注战略议题**（可选）：是否已有企业战略 / 年度重点，使环境要素与其关联？提供则校验相关性，不提供则标注「待提供战略后复核」，不卡流程（提示：不提供会导致"哪些要素更重要"的判断失真）。

**缺失信息处理（通用规则 3）：** 对无法获取且影响结果的信息，**向用户说明影响**，若用户执意不提供，在该部分明确标注「供参考·待确认」或「待提供战略后复核」。

## 三、输入载体判断（通用规则 2）

| 用户输入形式 | Agent 动作 |
|------|-----------|
| 纯对话描述 | 直接进入六门交互采集 |
| `.docx / .pdf / .xlsx / .md / .txt` 上传 | 调用文档解析技能（docx / markitdown / pdf 技能）提取文本，作为"待用户确认的环境要素草稿"，**逐条向用户核对**后再纳入——不得直接当作已确认内容 |
| 结构化 JSON（如历史分析结果） | 读取后转入"汇总与报告生成"阶段，并请用户确认数据时效性 |

## 四、核心机制：六门强制交互采集锁

把 PESTEL 拆成 **6 道必须依次过的采集门**。每道门未拿到用户**实质输入**就不推进下一门。

| 门 | 维度 | Agent 行为 | 锁规则 |
|----|------|-----------|--------|
| 门1 | **P** 政治 Political | 给思考维度+示例 → 强制提问索要 | 无实质输入 → 提示缺失，不进下一门 |
| 门2 | **E** 经济 Economic | 同上 | 同上 |
| 门3 | **S** 社会 Social | 同上 | 同上 |
| 门4 | **T** 技术 Technological | 同上 | 同上 |
| 门5 | **E** 环境 Environmental | 同上 | 同上 |
| 门6 | **L** 法律 Legal | 同上 | 同上 |

**Agent 可做的：**
- 给出每个维度的思考维度清单（见 `references/dimension_checklist.md`）；
- 给**示例**（明确标注"以下仅为示范，请基于贵方实际填写"）；
- 澄清用户表述、追问细节、提示"这条更像政治维度还是法律维度"（给归类建议）；
- 当用户把多个维度混在一起时，帮助拆分到正确象限。

**Agent 不可做的（硬性纪律）：**
- 替用户写 P/E/S/T/E/L 内容；
- 替用户判定某要素影响程度（高/中/低）或机会/威胁归属；
- 替用户定优先级。

**用户跳过某门：** 标注「供参考·待确认」并明确告知"将导致趋势影响评估与战略建议失真"，允许演示场景临时填充。

引导话术与每门示例见 `references/interaction_guide.md`。

## 五、汇总与战略映射（收集后，沿用本技能逻辑）

六门收齐后，Agent 按 `references/trend_logic.md` 自动推导（**标注每条结论的来源要素，交由用户确认**）：
1. **趋势汇总**：识别各维度关键趋势与变化方向；
2. **影响评估**：对每个要素评估"对分析对象的潜在影响方向"（提示性，最终判定权归用户）；
3. **机会 / 威胁初步映射**：将要素初步归为机会或威胁（标注建议性质）；
4. **战略议题关联**（若用户提供战略）：标注哪些要素与既定战略强相关；
5. **关键监测点**：建议后续需持续关注的外部环境信号。

---

## 六、输出形式（通用规则 4 + 双版本）

收集完成并汇总后，**先向用户展示报告大纲**（见 `assets/PESTEL分析_输出大纲.md`），确认结构后再生成：

- `PESTEL分析_报告.md`：标准结构化文档；
- `PESTEL分析_报告.html`：**精美网页版**（六维度卡片 + 趋势影响表 + 机会威胁映射 + 关键监测点，响应式 CSS），用于演示。

由 `scripts/build_report.py` 读取结构化 JSON 生成双版本，保证格式稳定。

## 七、工作流总览

```
[前置说明] → [确认分析对象/范围/战略(可选)]
   → 门1 P → 门2 E → 门3 S → 门4 T → 门5 E → 门6 L   （强制交互锁，逐一引导）
   → [汇总：趋势/影响/机会威胁映射/战略关联/监测点]
   → [展示报告大纲，确认]
   → [build_report.py 生成 MD + 精美HTML]
```

## 十、相关技能联动（战略分析工具链）

PESTEL 是工具链的**外部宏观入口**，与五力、VRIO 共同构成 SWOT 的上游输入。完整链路：

`pestel-analysis`（宏观外部）→ `porters-five-forces`（行业外部）→ `vrio-analysis`（内部）→ `swot-analysis`（收敛）→ `smart-goal-setter`（落成目标）→ `qms-objective-decomposition`（质量分解落地）

- **同链外部分析（互补）**：`porters-five-forces`（行业结构中观）——PESTEL 看宏观大环境，五力看行业竞争结构，二者叠加覆盖外部 O/T
- **内部对照**：`vrio-analysis` 提供内部 S/W，与 PESTEL 的 O/T 在 `swot-analysis` 汇合
- **收敛与落地**：PESTEL 扫出的机会/威胁 → `swot-analysis` 的 O/T 象限 → SWOT 战略 → `smart-goal-setter` 落成目标
- **关系说明**：PESTEL 不做行业结构深挖（那是五力的活），也不做内部评估（那是 VRIO 的活），聚焦宏观环境扫描；产物通常作为 SWOT 上游 feeder 被直接引用。
