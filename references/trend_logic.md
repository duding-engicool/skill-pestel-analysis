# PESTEL 趋势汇总与机会威胁映射逻辑

> 本文件定义"六门收齐后"Agent 如何汇总。所有推导**必须标注来源要素**，且**最终判定权归用户**。

## 一、趋势汇总

对每个维度，将用户提供的要素归纳为 1-3 条**趋势表述**（变化方向 + 信号）：
- 格式：`[维度] 趋势：<方向性描述>（依据：用户提供的要素 X、Y）`
- 示例：`E 经济趋势：目标市场消费力走弱（依据：用户提供的"失业率上升""可选消费下滑"）`
- Agent 仅做归纳整理，**不新增用户未提及的要素**。

## 二、影响评估（提示性）

对每个要素评估"对分析对象的潜在影响方向"，提供**提示性分级**：
- 影响方向：正向 / 负向 / 中性
- 提示强度：高 / 中 / 低（基于要素本身表述的确定性，非 Agent 主观放大）
- 标注：`（影响强度为提示性，请结合贵方实际确认）`
- **Agent 不替用户拍板"高/中/低"的最终结论**，仅呈现提示逻辑。

## 三、机会 / 威胁初步映射

将要素初步归为机会（O）或威胁（T）：
- 正向 / 可利用的外部环境 → 机会建议
- 负向 / 需防御的外部环境 → 威胁建议
- 标注：`（以下为初步映射建议，最终归类请用户确认）`
- 若用户提供战略议题，可在映射旁标注"与战略 X 强相关"。

## 四、战略议题关联（可选）

若用户提供企业战略 / 年度重点：
- 对每个要素标注是否与既定战略相关、关联方式；
- 未提供则整段标注「待提供战略后复核」，并提示"不提供会导致'哪些要素更重要'的判断失真"。

## 五、关键监测点

基于所有要素，提炼 5-10 个**后续应持续关注的外部环境信号**（如"某政策落地细则""某项技术商用时点"），供用户建立监测机制。

---

## 与 SWOT 的衔接（可选联动）

PESTEL 识别的机会 / 威胁，可喂给 `swot-analysis`：
- PESTEL 的 O/T 初步映射 → 作为 SWOT 门3（机会）、门4（威胁）的用户输入草稿；
- 衔接时明确告知用户："以下来自 PESTEL 扫描，请在 SWOT 中基于贵方内部能力进一步确认"。
